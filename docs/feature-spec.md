# 기능 명세

## 목적

아웃바운드 영업 담당자가 고객리스트를 가져와 전화, 문자, 길찾기, 방문/상담 이력을 빠르게 남길 수 있도록 돕는 PWA입니다.

## 대상 환경

- iPhone Safari
- iPhone 홈화면 추가 PWA
- iPad
- PC 브라우저
- iPhone/iPad 네이티브 앱 전환 대상

네이티브 앱 전환 시에는 `docs/native-pwa-parity-spec.md`와 `docs/ui-reference.md`를 함께 봅니다.

## 고객리스트 import

- CSV 파일을 읽습니다.
- iOS 네이티브 앱은 연락처에서 고객을 가져올 수 있습니다.
  - 개별 연락처 선택은 iOS 연락처 선택 화면을 사용합니다.
  - 그룹 선택은 앱 안에서 연락처 그룹 목록을 표시하고, 선택한 그룹의 연락처를 한 번에 가져옵니다.
  - 여러 그룹을 선택할 수 있으며, 같은 연락처가 여러 그룹에 있을 경우 한 번만 가져옵니다.
  - 이름, 전화번호, 우편주소, 회사/부서/직함 보조 정보를 고객 데이터로 변환합니다.
  - 저장 전 대상 고객리스트를 새로 만들지 기존 리스트에 추가할지 선택합니다.
  - 전화번호가 이미 앱의 고객 데이터에 있으면 기본적으로 건너뜁니다.
- 첫 행 헤더를 기준으로 열을 자동 인식합니다.
- 헤더가 없거나 불명확한 경우 사용자가 첫 행 헤더 사용을 끄고 열별 의미를 직접 지정합니다.
- 지원 필드:
  - 고객명
  - 연락처
  - 주소
  - 생년월일
  - 메모
  - 위도
  - 경도
- `핸드폰1`, `핸드폰2`처럼 숫자 접미사가 붙은 헤더도 연락처로 인식합니다.
- `우편물수령지`, `수령지`, `배송지`, `도로명주소`, `지번주소` 등 주소성 헤더를 주소로 인식합니다.
- 저장 전 매핑 결과를 확인할 수 있습니다.
- 저장된 고객리스트는 고객사 이름, 리스트 이름, 원본 파일명을 보관합니다.

## 고객리스트 수명주기

- 일상적인 제거는 `보관`을 기본으로 하며 고객과 전화·문자·방문·메모·사진·스케줄 기록을 유지합니다.
- 보관한 리스트는 `리스트 관리 > 보관함`에서 복원할 수 있습니다.
- 영구삭제는 별도 메뉴와 확인창으로 분리하고 삭제 전 고객, 터치, 방문, 사진, 스케줄, 캠페인 건수를 표시합니다.
- 영구삭제는 앱 저장소의 연결 데이터와 사진·지도·음성 파일을 제거하지만 iPhone 연락처 앱의 연락처는 삭제하지 않습니다.
- 리스트를 영구삭제하면 tombstone을 남겨 Google Drive 동기화 시 다른 기기의 오래된 목록이 다시 살아나지 않게 합니다.
- 리스트의 고객 전체를 다른 활성 리스트로 이동하면 연결된 방문·메모·사진·스케줄·캠페인·상태 기록도 함께 이동합니다.

## 관리 기간과 활동 보고서

- 사용자는 `7~9월`, `2026년 하반기`, 특정 영업 캠페인처럼 이름, 시작일, 종료일, 대상 고객리스트를 가진 관리 기간을 생성할 수 있습니다.
- 관리 기간은 진행 중, 마감, 보관 상태를 가지며 마감할 때 요약 메모를 남길 수 있습니다.
- 관리 기간 보고서는 기간 경계를 시작일 00:00부터 종료일 23:59:59까지 포함해 조회합니다.
- `내 활동`은 관리 기간 없이도 시작일, 종료일, 고객리스트를 직접 선택해 조회할 수 있습니다.
- 보고서는 기존 전화·문자·방문·텍스트 메모·사진 메모·음성 메모 원본 로그와 고객/리스트/스케줄/대시보드 관리 이벤트를 하나의 시간순 활동 흐름으로 결합합니다.
- 같은 전화·문자·방문 기록을 별도 보고서용 테이블에 복사하지 않아 중복 집계와 원본 불일치를 방지합니다.
- 요약에는 전체 활동 수, 활동 고객 수, 전화, 문자, 방문, 메모 건수를 표시합니다.
- 앱 외부의 일반 전화 앱 통화내역과 메시지 앱 수발신 내역은 iOS 공개 API로 읽을 수 없으므로 자동 통합하지 않습니다.
- 설계 참고: Pipedrive의 활동 보고서 필터/요약 구조와 HubSpot의 삭제 복구 정책처럼 활동 조회와 데이터 수명주기를 분리합니다.
  - https://support.pipedrive.com/en/article/insights-reports-activities-performance
  - https://knowledge.hubspot.com/records/restore-deleted-records

## 방문 히스토리

- 고객카드와 지도 팝업의 `방문` 버튼은 고객 상태를 완료로 바꾸지 않고 방문 히스토리만 남깁니다.
- 고객 상태 `완료`는 고객 상세 화면의 `상태와 스케줄 > 완료 처리`에서만 변경합니다.
- 방문 버튼을 누르면 `상세한 히스토리를 기록하겠습니까?`를 표시합니다.
- `아니요`를 선택하면 현재 시각과 현재 위치의 역변환 주소를 저장하고, 지도 스냅샷 썸네일을 함께 남깁니다.
- 방문 히스토리에는 위도/경도를 저장하지 않습니다.
- 지도 썸네일을 터치하면 저장된 주소 문자열로 Apple 지도를 엽니다.
- `예`를 선택하면 `텍스트 메모`, `사진 메모`, `음성 메모` 중 하나를 기록합니다.
- 고객 상세 화면의 `고객 터치` 섹션에서 전화, 문자, 길찾기, 사진 메모, 음성 메모, 텍스트 메모, 방문을 실행합니다.
- 고객카드에서는 수정 대신 음성 메모를 바로 실행하고, 고객 정보 수정은 고객 상세 화면에서 수행합니다.
- 고객 상세 화면은 최근 사진 메모, 음성 메모, 텍스트 메모를 각각 표시합니다.
- 음성 메모 입력 화면은 사진 메모처럼 하단에 최근 음성 메모를 표시합니다.
- 음성 메모 상세에서는 전사 텍스트와 저장된 음성을 함께 확인할 수 있습니다.
- 음성 메모 상세에서는 재생 슬라이더로 빠르게 구간을 탐색할 수 있습니다.
- 전사 구간이 생성된 음성 메모는 전사 본문 안의 단어/구간을 터치해 해당 시점부터 음성을 재생할 수 있습니다.
- 음성 메모 재생 중 현재 전사 단어/구간은 전사 본문 안에서 하이라이트로 표시합니다.
- 전사 구간은 새 음성 메모 저장 후 온디바이스 전사가 완료되거나, 기존 음성 메모 상세 진입 시 재전사가 완료되면 표시됩니다.
- 녹음 중에는 일시정지, 다시 녹음, 녹음 종료를 지원합니다.
- 화면 잠금 중에도 녹음이 이어질 수 있도록 iOS 백그라운드 오디오 모드를 사용합니다.
- 전화 수신 자체를 앱이 막거나 벨소리를 강제로 무음 처리하는 것은 iOS 정책상 지원하지 않습니다. 전화 등 오디오 인터럽트가 발생하면 녹음을 일시정지하고 사용자가 다시 이어서 녹음하도록 안내합니다.
- 텍스트 메모 상세에서는 전체 텍스트를 확인할 수 있습니다.
- 음성 메모는 앱 내부 저장소에 저장하고, Apple Speech 기반 온디바이스 전사를 비동기로 시도합니다.
- 전사 결과는 `전사 대기중`, `전사중`, `완료`, `실패` 상태로 히스토리에 표시합니다.
- Google Drive 백업/복원에는 지도 스냅샷과 음성 메모 파일을 포함합니다.

## OCR 기반 고객리스트 import 계획

- 스캔 이미지 파일을 입력받아 인쇄된 고객리스트의 셀 값을 OCR로 읽는 기능을 검토합니다.
- 원본 표 디자인을 완전히 복원하지 않고, 행/열 데이터로 변환하는 것을 목표로 합니다.
- 사용자가 각 열 이름을 직접 지정합니다.
  - 예: 1열 = 고객명, 2열 = 연락처, 3열 = 주소, 4열 = 메모
- 1차 검증은 Mac에서 이미지 파일 입력 방식으로 진행합니다.
- iPhone/iPad 확장 시 문서 스캔은 VisionKit, OCR은 Apple Vision 사용을 검토합니다.
- Android 확장 시 문서 스캔은 ML Kit Document Scanner, OCR은 ML Kit Text Recognition v2 사용을 검토합니다.
- 상세 개발 계획은 `docs/customer-list-ocr.md`에서 관리합니다.
- 표 선이 명확한 문서는 격자 기반 셀 경계를 우선 검토하고, 선이 없거나 신뢰도가 낮은 문서는 OCR 좌표 기반 복원으로 자동 전환합니다.
- 이미지 방향은 OCR 구조로 먼저 확정한 후 격자 분석을 실행해 180도 반대 방향이 선택되는 것을 방지합니다.
- 격자 검출이 실패해도 반복 텍스트의 기울기를 추정하고, 반복 열의 위아래 순서를 이용해 원근·구김으로 갈라진 행을 복원합니다.
- 촬영 이미지에서는 문서 사각형을 검출해 원근을 편 후보와 원본 후보를 함께 평가하며, 회전 전후에서 가장 안정적인 결과를 선택합니다.
- 전화·날짜·금액 본문 신호가 있는 첫 행은 자동 헤더로 제거하지 않고 데이터로 유지합니다.
- 한 셀에 여러 행의 본문 형식이 반복되면 행 병합 후보로 감점하고, 격자 보정으로 정상 행이 사라지는 결과는 제한합니다.
- 연속 번호가 있는 본문은 증가 방향을 우선하며, 반복 본문 밖의 촬영 배경 문자는 CSV 데이터에서 제외합니다.
- 빈 셀만 존재하는 열은 사진 품질에 따라 자동 확정할 수 없으므로 저장 전 열 매핑 화면에서 사용자가 확인합니다.

## 고객 관리

- 고객 직접 추가
- CSV 없이 빈 고객리스트 직접 생성
- 수동 생성한 고객리스트에 고객 직접 추가
- 가져온 고객을 iOS 연락처에 일괄 등록
  - 고객리스트 이름과 같은 연락처 그룹/리스트 생성
  - 기본값은 고객명 앞에 `#` 접두어를 붙인 이름으로 등록
  - `#` 접두어는 카카오톡 자동등록 방지 목적의 기본값으로 사용
  - 사용자는 접두어를 붙일지 여부를 선택할 수 있고 접두어/접미어를 수정할 수 있음
  - 중복 연락처 검사 후 업데이트/건너뛰기 선택
  - 가져오기 저장 직후 또는 기존 고객리스트 상세에서 실행 가능
- 등록 연락처 관리
  - 연락처 그룹은 원본 연락처와 분리된 참조 목록이므로 `그룹만 삭제`와 `앱 생성 연락처와 그룹 삭제`를 구분합니다.
  - 앱이 신규 생성한 연락처만 영구 삭제할 수 있으며 기존 연락처 업데이트/중복 연결 항목은 항상 보호합니다.
  - 다른 연락처 그룹에도 속한 항목과 등록 후 이름 또는 전화번호가 바뀐 항목은 자동 제외합니다.
  - 삭제 전 실제 삭제 수, 기존 연락처 보호 수, 다른 그룹 소속, 변경/누락 수를 표시하고 최종 확인을 받습니다.
  - 그룹이 이미 수동 삭제된 과거 등록분은 저장된 연락처 식별자, 등록 이름과 전화번호가 모두 현재 연락처와 일치할 때만 삭제 후보로 인정합니다.
  - 연락처 식별자는 현재 앱 설치에 귀속하므로 다른 기기에서 복원한 등록 작업을 자동 삭제에 사용하지 않습니다.

- 고객 완료 처리
- 완료 취소

- 문자 시도 기록
- 단체문자 기능은 iOS Shortcuts 연동 방식으로 개발을 검토합니다.
  - 여러 수신자를 한 문자 작성창에 넣어 단톡방을 만들지 않습니다.
  - 앱에서 대상 고객과 개인화 메시지를 준비하고, Shortcuts가 1명씩 순차 발송하도록 위임합니다.
  - 실제 고객 대상 발송은 현재 고객리스트, 오늘 스케줄, 현재 검색 결과를 대상으로 선택할 수 있습니다.
  - 초기 검증용으로 사용자의 테스트 번호 1~2개를 반복 대상으로 만드는 테스트 모드도 유지합니다.
  - 고객별 제외 토글과 중복 전화번호 제거 옵션을 제공합니다.
  - `{고객명}`, `{이름}`, `{연락처}`, `{주소}`, `{메모}`, `{순번}`, `{전체}` 치환을 지원합니다.
  - 메시지 사이에는 랜덤 지연 시간을 적용하고, 일정 인원 단위로 묶음 휴식 시간을 둘 수 있습니다.
  - 딜레이 꺼짐/고정/랜덤을 선택할 수 있습니다.
  - 한 캠페인에 사진 여러 장과 일반 파일 여러 개를 첨부할 수 있으며 사진과 파일을 함께 선택할 수 있습니다.
  - 첨부 목록은 선택 순서, 파일명, 형식과 크기를 표시하고 개별 삭제와 순서 변경을 지원합니다.
  - 앱 안에서 iCloud 단축어 공유 링크를 열어 설치 화면으로 이동하고, `SoheeGroupSMS` 단축어 열기 테스트를 제공합니다.
  - iOS 정책상 단축어는 사용자가 Shortcuts 앱에서 직접 추가해야 하며 앱이 조용히 자동 설치하지 않습니다.
  - `SoheeGroupSMS` 단축어 상세 설계는 `docs/group-sms-shortcut.md`에서 관리합니다.
  - 제품 UI는 자동화 준비, 대상 선택, 메시지 작성, 발송 전 점검, 발송 진행, 결과와 복구의 6단계로 분리합니다.
  - 단축어 URL이 열렸다는 사실만으로 준비 완료 처리하지 않고, 본인 번호 실제 수신 시험과 시험한 단축어 버전을 별도로 기록합니다.
  - 대상 제외는 사용자 제외, 번호 없음/오류, 중복 번호, 최근 발송으로 구분해 수량과 목록을 표시합니다.
  - 현재 검증용 `GroupSmsTestView`의 긴 단일 Form은 진단 화면으로 유지하고 실제 고객용 캠페인 화면으로 사용하지 않습니다.
  - 실제 고객용 화면은 이동 중 조작 단계를 줄이기 위해 `대상`, `메시지`, `확인` 세 단계로 표시합니다. 자동화 준비는 상단 상태 배너, 발송 진행과 결과·복구는 실행 후 상태 화면으로 제공해 전체 6단계 제품 흐름을 유지합니다.
  - 단체문자는 별도 하단 탭을 차지하지 않습니다. 고객 탭에서는 현재 고객리스트, 오늘 탭에서는 오늘 스케줄을 기본 대상으로 바로 진입하며 캠페인 안에서 검색 결과 범위로 변경할 수 있습니다.
  - 자동화 설치·버전·권한 진단은 기본 작성 흐름을 가리지 않도록 공구 아이콘의 별도 진단 화면에서 처리합니다.
  - 주요 전진·발송 버튼은 한 손 조작을 위해 화면 하단에 고정하고 최소 54pt 높이를 확보합니다.
  - 발송 대상에는 고객리스트, 오늘 스케줄, 검색 결과뿐 아니라 연락처 개인 다중 선택과 연락처 그룹 일괄 선택을 포함합니다.
  - 대상 단계는 현재 범위와 추가 연락처를 한 번에 `전체 선택` 또는 `전체 해제`할 수 있어야 하며, 전체 해제 후 필요한 고객만 개별 선택할 수 있어야 합니다.
  - 연락처 대상은 명시적으로 저장을 선택하지 않는 한 고객 데이터로 가져오지 않고 캠페인 수신자 스냅샷으로만 보관합니다.
  - 서로 다른 출처에서 같은 전화번호가 선택되면 전화번호 정규화 후 한 명만 발송 대상으로 유지합니다.

  - 예약은 해당 시각에 로컬 알림을 표시하고 사용자가 `발송 시작`을 확인하면 Shortcuts를 실행하는 방식입니다. iOS가 보장하지 않는 앱 단독 무인 백그라운드 발송으로 표현하지 않습니다.

  - 향후 별도 유료 단체문자 앱으로 분리할 수 있도록 캠페인 엔진은 CRM의 `Customer`와 저장 상태에 직접 의존하지 않게 구성합니다.

  - 미방문
  - 완료
  - 전체
  - 나이별

## 지도

- 고객 좌표를 지도에 표시합니다.
- 좌표가 없으면 OpenStreetMap Nominatim으로 주소 검색을 수행합니다.
- 중복 주소는 한 번만 검색합니다.
- 무료 API 안정성을 위해 주소 검색은 1초 간격을 유지합니다.
- 지도에는 핀 대신 고객 이름 라벨을 표시합니다.
- 라벨에는 완료/미완료/예정 상태를 반영합니다.
- 라벨 터치 시 작은 고객 카드 팝업을 표시합니다.
- 팝업에서 전화, 문자, 길찾기, 히스토리, 완료, 완료 취소, 스케줄 추가를 실행할 수 있습니다.

## 주소 처리

- 지역 그룹화는 도로명까지만 사용합니다.
  - 예: `산성대로 123`은 그룹 기준으로 `산성대로`
- 지도 검색과 티맵 전달 주소는 `도로명 + 건물번호`까지만 사용합니다.
- 상세 호수, 괄호 내용, 고객명은 길찾기 주소에서 제외합니다.

## 전화/문자/길찾기

- 전화:
  - `tel:` 링크로 전화앱 실행
- 문자:
  - `sms:` 링크로 문자앱 실행
  - 고객 전화번호를 정리해 현재 고객 번호만 전달
  - 템플릿 문자는 본문을 클립보드에 복사 후 문자앱 실행
  - 사용자 문자는 본문 자동 입력 없이 문자앱 실행
- 길찾기:
  - 티맵 앱 실행
  - 좌표가 있으면 좌표 기반 경로 요청
  - 좌표가 없으면 정규화 주소 검색

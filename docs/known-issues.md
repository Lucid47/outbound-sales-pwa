# 알려진 이슈와 해결 기록

## Group SMS 결과와 플랫폼 제약

상태: iOS 핵심 흐름 실기기 검증, 고도화 진행 중

- 2026-07-12 실제 iPhone에서 여러 번호와 반복 횟수 증가, 건별 확인 없는 자동 순차 발송을 검증했습니다.
- 앱은 Shortcuts에 발송을 요청하고 완료 callback을 받을 수 있지만 통신사 최종 도달을 직접 확인할 수 없습니다.
- 단축어의 `알림 보기`가 반복 안에 남아 있으면 발송마다 확인이 필요합니다. 운영 단축어에서는 제거합니다.
- 딜레이, callback, 사진/파일 첨부는 별도 실기기 검증이 필요합니다.
- Android 자동 SMS는 `SEND_SMS`와 Google Play의 제한 권한 정책 영향을 받으므로 iOS 단축어 구현을 그대로 포팅할 수 없습니다.

## iPhone 문자 자동 발송 불가

상태: 제약으로 문서화

내용:

- PWA에서는 사용자 개입 없는 SMS 자동 발송이 불가능합니다.
- 문자 본문 자동 삽입도 iOS에서 안정적으로 보장되지 않습니다.

현재 대응:

- 템플릿 문자는 본문을 클립보드에 복사합니다.
- 문자앱은 `sms:` 링크로 실행합니다.
- 실제 발송 여부가 아니라 문자 버튼을 누른 시도 이력을 저장합니다.

## 문자앱 실행 시 이전 고객 번호가 들어가는 문제

상태: 개선 완료

원인 후보:

- iOS 문자앱이 이전 작성창을 재사용하는 동작
- 고객 카드 클릭 이벤트와 문자 버튼 클릭 이벤트가 함께 처리될 가능성
- `sms:` 링크 직접 이동이 브라우저 상태와 섞일 가능성

개선:

- 현재 고객 번호를 매번 정리해 `sms:` 링크 생성
- `window.location.href` 직접 변경 대신 새 링크를 만들어 실행
- 전화/문자/길찾기 버튼의 이벤트 전파 차단
- 문자 시트에서는 문자앱 실행 후 시트를 닫도록 순서 조정

## 모바일 카드형/목록형 전환이 안 되는 것처럼 보이는 문제

상태: 개선 완료

원인:

- 전환 로직은 작동했지만 모바일 CSS에서 카드형과 목록형의 시각 차이가 작았습니다.

개선:

- 모바일 목록형 전용 스타일 추가
- 목록형에서 고객 카드가 더 압축된 리스트 형태로 표시되도록 변경
- 주소와 보조 정보 말줄임 처리
- 버튼 높이와 간격 조정

## GitHub Pages 배포 간헐 실패

상태: 운영 대응 방식 정리

증상:

```text
Deployment failed, try again later.
```

대응:

- 앱 빌드 실패가 아니라 Pages 배포 서비스의 일시 오류로 판단
- 실패한 작업만 재실행하면 정상 배포됨

## 지도 위치 누락

상태: 일부 개선, 지속 관찰

원인 후보:

- 상세 주소, 동호수, 괄호 설명, 지번 보조 정보가 주소 검색에 섞임
- OpenStreetMap의 한국 도로명주소 검색 정확도 한계

개선:

- 지도 검색 주소는 `도로명 + 건물번호`까지만 사용
- 지역 그룹화는 도로명까지만 사용
- 중복 주소 검색 제거
- 위치 표시가 안 된 고객과 원인을 지도 하단에 표시

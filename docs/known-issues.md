# 알려진 이슈와 해결 기록

## 일반 전화·문자 앱 기록 자동 통합 제한

상태: iOS 플랫폼 제약으로 문서화

- iOS 공개 API는 다른 앱의 통화 기록이나 메시지 앱의 SMS/iMessage 수발신 내역을 제3자 앱에 제공하지 않습니다.
- 단축어도 시스템의 최근 통화 목록을 직접 읽어 앱으로 복사할 수 없습니다.
- 따라서 `소희야 가자`에서 실행하거나 사용자가 앱 안에서 직접 남긴 전화·문자·방문·메모 기록만 활동 보고서에 자동 반영합니다.
- 외부에서 처리한 활동은 텍스트 메모 또는 방문 기록으로 수동 보완할 수 있습니다.

## 고객리스트·고객 영구삭제와 기기 연락처

상태: 분리 정책 구현

- 고객리스트의 기본 제거는 복원 가능한 보관입니다.
- 영구삭제는 앱 내부 고객과 연결 기록 및 앱 전용 파일을 제거하지만 iPhone 연락처는 삭제하지 않습니다.
- 연락처 삭제가 필요하면 고객 탭의 별도 `연락처 관리` 기능에서 앱이 생성한 연락처인지 확인한 뒤 실행해야 합니다.
- 고객리스트와 개별 고객 tombstone은 Drive 병합·복원에 반영됩니다.
- 방문·메모처럼 고객 아래의 개별 기록만 삭제하는 범용 tombstone은 추가 구현 항목입니다.

## 연락처 그룹 삭제 후 연락처가 남는 현상

상태: 안전 삭제 기능 구현, 실제 iPhone 회귀 시험 필요

- iOS 연락처 그룹은 연락처 원본을 소유하지 않는 참조 목록이므로 그룹 삭제만으로 연락처는 삭제되지 않습니다.
- 고객 탭의 `연락처 관리`에서 `그룹만 삭제` 또는 `앱 생성 연락처와 그룹 삭제`를 선택할 수 있습니다.
- 앱이 신규 생성했다고 확인되는 연락처만 삭제하며 기존 연락처, 다른 그룹 소속, 등록 후 변경된 항목은 자동 제외합니다.
- 그룹을 이미 수동 삭제한 과거 등록분은 연락처 식별자·등록 이름·전화번호가 모두 일치할 때만 삭제합니다.
- 연락처 삭제는 iCloud 또는 Google 연락처 계정과 다른 기기에 동기화될 수 있으므로 삭제 전 미리보기와 최종 확인을 거칩니다.

## Group SMS 결과와 플랫폼 제약

상태: iOS 핵심 흐름 실기기 검증, 고도화 진행 중

- 2026-07-12 실제 iPhone에서 여러 번호와 반복 횟수 증가, 건별 확인 없는 자동 순차 발송을 검증했습니다.
- 앱은 Shortcuts에 발송을 요청하고 완료 callback을 받을 수 있지만 통신사 최종 도달을 직접 확인할 수 없습니다.
- iOS 공개 API로 Shortcuts를 화면 전환 없이 실행할 수는 없습니다. 지원 흐름은 Shortcuts로 전환한 뒤 완료 callback으로 앱에 자동 복귀하는 방식입니다.
- callback scheme이 누락된 일반 빌드에서도 자동 복귀하도록 앱 전용 URL scheme을 명시 등록했습니다. callback이 유실된 경우 진행 화면의 복구 메뉴에서 요청 상태를 정리할 수 있습니다.
- 단축어의 `알림 보기`가 반복 안에 남아 있으면 발송마다 확인이 필요합니다. 운영 단축어에서는 제거합니다.
- 딜레이, callback, 사진/파일 첨부는 별도 실기기 검증이 필요합니다.
- 예약 발송은 지정 시각의 로컬 알림을 통해 사용자가 발송을 확인하는 방식입니다. iOS 백그라운드 작업은 정확한 실행 시각을 보장하지 않고 앱이 Shortcuts를 임의로 전면 실행할 수 없으므로 앱 단독 무인 예약 발송은 지원하지 않습니다.
- 활성 예약 알림은 예약을 생성한 기기에만 등록됩니다. Drive에서 다른 기기로 복원한 예약 데이터는 중복 발송 방지를 위해 그 기기의 실행 예약으로 자동 활성화하지 않습니다.
- Android 자동 SMS는 `SEND_SMS`와 Google Play의 제한 권한 정책 영향을 받으므로 iOS 단축어 구현을 그대로 포팅할 수 없습니다.

## iPhone 문자 자동 발송 불가

상태: 제약으로 문서화

내용:

- PWA에서는 사용자 개입 없는 SMS 자동 발송이 불가능합니다.
- 문자 본문 자동 삽입도 iOS에서 안정적으로 보장되지 않습니다.

현재 대응:

- 템플릿 문자는 본문을 클립보드에 복사합니다.
- 문자앱은 `sms:` 링크로 실행합니다.
- 실제 발송 여부가 아니라 문자 버튼을 누른 시도 이력을 저장합니다.

## 문자앱 실행 시 이전 고객 번호가 들어가는 문제

상태: 개선 완료

원인 후보:

- iOS 문자앱이 이전 작성창을 재사용하는 동작
- 고객 카드 클릭 이벤트와 문자 버튼 클릭 이벤트가 함께 처리될 가능성
- `sms:` 링크 직접 이동이 브라우저 상태와 섞일 가능성

개선:

- 현재 고객 번호를 매번 정리해 `sms:` 링크 생성
- `window.location.href` 직접 변경 대신 새 링크를 만들어 실행
- 전화/문자/길찾기 버튼의 이벤트 전파 차단
- 문자 시트에서는 문자앱 실행 후 시트를 닫도록 순서 조정

## 모바일 카드형/목록형 전환이 안 되는 것처럼 보이는 문제

상태: 개선 완료

원인:

- 전환 로직은 작동했지만 모바일 CSS에서 카드형과 목록형의 시각 차이가 작았습니다.

개선:

- 모바일 목록형 전용 스타일 추가
- 목록형에서 고객 카드가 더 압축된 리스트 형태로 표시되도록 변경
- 주소와 보조 정보 말줄임 처리
- 버튼 높이와 간격 조정

## GitHub Pages 배포 간헐 실패

상태: 운영 대응 방식 정리

증상:

```text
Deployment failed, try again later.
```

대응:

- 앱 빌드 실패가 아니라 Pages 배포 서비스의 일시 오류로 판단
- 실패한 작업만 재실행하면 정상 배포됨

## 지도 위치 누락

상태: 일부 개선, 지속 관찰

원인 후보:

- 상세 주소, 동호수, 괄호 설명, 지번 보조 정보가 주소 검색에 섞임
- OpenStreetMap의 한국 도로명주소 검색 정확도 한계

개선:

- 지도 검색 주소는 `도로명 + 건물번호`까지만 사용
- 지역 그룹화는 도로명까지만 사용
- 중복 주소 검색 제거
- 위치 표시가 안 된 고객과 원인을 지도 하단에 표시

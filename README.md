# 아웃바운드 영업 도우미 PWA

아이폰에서 무료로 사용할 수 있도록 만든 아웃바운드 영업 지원 PWA입니다.

PWA는 Progressive Web App의 약자로, 웹으로 배포하지만 아이폰 Safari에서 홈 화면에 추가하면 별도 앱처럼 실행할 수 있습니다. 현재 운영 주소는 다음과 같습니다.

```text
https://lucid47.github.io/outbound-sales/
```

## 개발 목표

- 외부 고객사에서 받은 비정형 CSV 고객리스트를 가져와 저장한다.
- 운전 중에도 큰 버튼과 단순한 탭 구조로 빠르게 전화, 문자, 길찾기, 기록을 남긴다.
- 고객별 터치 이력, 완료 상태, 메모, 스케줄을 계속 보관한다.
- 서버 비용 없이 아이폰 브라우저의 로컬 저장소를 중심으로 운영한다.
- 홈 화면에 추가한 뒤 다시 실행해도 기존 데이터가 자동으로 불러와지게 한다.

## 현재 구현 상태

### 고객리스트 가져오기

- CSV 파일 import 지원
- 첫 행 헤더를 보고 고객명, 연락처, 주소, 생년월일, 메모, 좌표 열 자동 인식
- `성명`, `우편물주소`, `핸드폰1`, `핸드폰2`, `비고` 같은 실제 고객사 CSV 헤더 대응
- `핸드폰1`처럼 헤더 끝에 숫자가 붙어도 `핸드폰` 계열로 인식
- 고객사 이름과 고객리스트 이름은 사용자가 직접 입력
- import한 고객리스트별로 고객, 스케줄, 로그를 분리 저장
- 고객리스트 삭제 시 해당 리스트의 고객, 로그, 스케줄도 함께 삭제

### 고객 관리

- 고객 탭에서 전체 고객 확인
- 고객 수동 등록
- 고객 이름, 연락처, 주소, 생년월일, 메모 수정
- 완료 처리와 완료 취소 지원
- 남은 고객, 터치 고객, 완료 고객 카운트 표시
- 카운트 카드 터치 시 해당 고객 이름과 주소를 작은 팝업으로 확인
- 고객 검색 지원
  - 이름 일부
  - 전화번호 일부
  - 주소 일부
  - 공백을 제외한 주소 조각

### 오늘 스케줄

- 오늘 만날 고객을 미리 스케줄에 추가
- 스케줄 고객을 카드 형태로 표시
- 특정 순서만 강제하지 않고 랜덤하게 방문 확인 가능
- 고객 완료 처리 시 스케줄 상태도 완료로 변경
- 스케줄에서 고객 삭제 가능
- 가까운 순 정렬 지원

### 지도

- 고객 탭과 지도 화면에서 고객 위치 표시
- 핀 대신 고객 이름 라벨 형태로 표시
- 완료, 예정, 미완 상태를 지도 라벨에 함께 표시
- 지도 라벨 터치 시 작은 고객 카드 팝업 표시
- 팝업에서 전화, 문자, 길찾기, 이력, 완료, 완료 취소, 스케줄 추가 가능
- 내 위치 버튼 지원
- 내 위치를 지도에 별도 핀으로 표시
- 지도 하단에 위치 표시가 안 된 고객과 간단한 원인 표시
- 지도 표시용 주소는 도로명주소 원칙을 반영해 `도로명 + 건물번호` 중심으로 정규화

### 위치와 주소 처리

- 주소 그룹화는 넓은 구분을 위해 도로명까지만 사용
  - 예: `산성대로 123`은 그룹 기준으로 `산성대로`
- 지도 검색과 티맵 전달 주소는 `도로명 + 건물번호`까지만 사용
  - 상세 호수, 괄호 안 설명, 고객명은 길찾기 주소에서 제외
- OpenStreetMap Nominatim으로 주소를 좌표 변환
- 같은 주소는 중복 검색하지 않고 한 번 검색한 결과를 같은 주소 고객에게 공유
- 검색 간격은 무료 API 안정성을 위해 1초 유지

### 전화, 문자, 길찾기

- 전화 버튼: `tel:` 링크로 전화앱 실행
- 문자 버튼: `sms:` 링크로 문자앱 실행
- 연락처가 비어 있거나 너무 짧으면 문자/전화앱을 열지 않고 안내 메시지 표시
- 문자앱 실행은 iOS 제한을 줄이기 위해 사용자의 터치 직후 즉시 수행
- 전화/문자 시도 자체를 고객별 히스토리에 기록
- 문자 템플릿 지원
- 기본 템플릿 수정 가능
- 템플릿 문자는 본문을 클립보드에 복사한 뒤 문자앱을 열도록 구현
- 사용자 문자보내기는 본문 자동입력 없이 문자앱만 실행
- 티맵 길찾기 연결 지원
- 티맵으로 고객 이름을 넘기지 않고 정규화된 주소만 전달
- 길찾기 요청 후 원래 PWA로 복귀할 수 있도록 별도 티맵 요청 페이지에 머무르는 문제 개선

### 기록과 히스토리

- 기록 탭에서 고객별 히스토리 확인
- 기록 탭에서도 검색 지원
  - 이름, 전화번호, 주소 일부 검색 가능
- 활동 기록 화면에서 방문 로그, 문자 로그, 전화 로그, 메모 확인
- 고객별 상세 히스토리 팝업 제공
- 메모 프리셋 제공
  - 전화하였으나 받지 않음
  - 문자로 연락함
  - 방문하였으나 부재
  - 사용자 템플릿
- 완료 고객은 음영/하이라이트로 구분
- 완료 고객도 다시 활성화 가능

### 백업과 데이터 보관

- 데이터는 서버가 아니라 아이폰 브라우저 IndexedDB에 저장
- 앱 재실행, 새로고침, 홈 화면 실행 후에도 기존 데이터 자동 로드
- 브라우저 저장소 영구 보관 요청 사용
- 백업 미실행 또는 오래된 백업에 대한 안내 메시지 표시
- JSON 백업 내보내기와 복원 지원
- Google Drive 앱데이터 기반 동기화 1단계 구현
- Google Drive 일반 파일 백업 내보내기 1단계 구현
- Safari 데이터 삭제, 기기 변경, 브라우저 캐시 정리 시 데이터 손실 가능성이 있으므로 주기적 백업 필요

### Google Drive 동기화

추천 구조는 `앱데이터 동기화 + 일반 Drive 백업 내보내기` 조합입니다.

```text
기본 동기화: Google Drive appDataFolder
사용자 백업: 일반 Google Drive JSON 파일
로컬 저장: IndexedDB
```

1단계 구현 범위:

- 설정 화면에 Google Drive 동기화 섹션 추가
- Google Identity Services로 Drive API access token 요청
- `drive.appdata` 권한으로 앱 전용 숨김 동기화 파일 생성
- `outbound-sales-sync.json` 파일에 현재 앱 데이터를 저장
- Drive 데이터와 로컬 데이터를 ID 기준으로 병합
- 같은 ID가 양쪽에 있으면 더 최근 수정 시각의 데이터를 우선
- 로그성 데이터는 ID 기준으로 합쳐 데이터 유실을 줄임
- `drive.file` 권한으로 사용자가 볼 수 있는 JSON 백업 파일 생성

아직 1단계에서 제한되는 것:

- 삭제 동기화는 별도 `deletedAt` 설계가 필요함
- 앱이 완전히 닫힌 상태의 백그라운드 자동 동기화는 보장하지 않음
- Google OAuth Client ID 설정 전에는 버튼이 비활성화됨

## 주요 제약

### iPhone 문자 기능 제약

PWA에서는 iOS 정책상 문자 자동 발송이 불가능합니다.

가능한 것:

```text
전화번호가 들어간 문자앱 열기
템플릿 문구를 클립보드에 복사
사용자가 문자앱에서 붙여넣고 직접 전송
문자 시도 로그 저장
```

불가능하거나 안정적으로 보장하기 어려운 것:

```text
사용자 개입 없는 SMS 자동 전송
문자 본문을 iOS 문자앱에 항상 자동 삽입
수신자별 개별 문자 자동 발송
실제 발송 성공 여부 확인
실제 통화 연결 여부 확인
```

현재 앱의 기록은 실제 발송/통화 성공 여부가 아니라 사용자가 전화 또는 문자 버튼을 눌렀다는 터치 이력입니다.

### 지도와 주소 제약

- OpenStreetMap 무료 검색은 한국 도로명주소 정확도가 완벽하지 않을 수 있습니다.
- 주소가 너무 상세하거나 괄호, 동호수, 별칭이 섞이면 검색 실패 가능성이 있습니다.
- 이 문제를 줄이기 위해 지도 검색과 티맵 전달 주소는 `도로명 + 건물번호`까지만 사용합니다.
- 대량 주소 검색은 무료 API 정책을 고려해 1초 간격으로 처리합니다.

## Google Drive 설정

Google Drive 동기화를 활성화하려면 Google Cloud에서 OAuth Client ID를 만들어야 합니다.

권장 설정:

```text
Google Cloud 프로젝트 생성
→ Google Drive API 활성화
→ OAuth 동의 화면 설정
→ OAuth Client ID 생성
→ Application type: Web application
→ Authorized JavaScript origins 등록
```

로컬 개발 origin:

```text
http://localhost:5173
```

배포 origin:

```text
https://lucid47.github.io
```

로컬 `.env` 예시:

```bash
VITE_GOOGLE_CLIENT_ID=your-google-oauth-web-client-id.apps.googleusercontent.com
```

GitHub Pages 배포에서는 저장소 Variables에 다음 값을 등록합니다.

```text
VITE_GOOGLE_CLIENT_ID
```

## 배포

현재 배포 방식은 GitHub Pages입니다.

```text
main 브랜치 push
→ GitHub Actions 실행
→ pnpm build
→ GitHub Pages artifact 업로드
→ https://lucid47.github.io/outbound-sales/ 에 반영
```

GitHub Pages 배포 단계에서 간헐적으로 `Deployment failed, try again later.` 오류가 발생한 적이 있습니다. 이 경우 앱 빌드 실패가 아니라 Pages 배포 서비스의 일시 오류였고, 실패한 작업만 재실행하면 정상 배포되었습니다.

## 최근 확인된 문제와 개선

### 문자보내기가 안 된다는 사용자 피드백

증상:

```text
문자 로그는 남아 있으나 문자앱 실행이 되지 않는다는 피드백 발생
```

가능 원인:

```text
CSV에서 연락처 열이 자동 인식되지 않아 고객 연락처가 비어 저장됨
iOS/PWA에서 사용자 터치 직후가 아닌 비동기 작업 후 sms: 링크를 열면 차단될 수 있음
```

개선:

```text
핸드폰1, 핸드폰2 같은 숫자 접미사 헤더 자동 인식
문자/전화 실행 전 연락처 유효성 확인
문자/전화 앱을 먼저 열고 로그 저장은 뒤에서 처리
문자 팝업에 연락처 없음 상태 표시
```

### 기록 탭 검색 미지원

증상:

```text
고객 탭에서는 검색되지만 기록 탭에서는 검색되지 않음
```

개선:

```text
기록 탭 상단에도 검색창 표시
전체 고객, 터치 고객, 완료 고객 카운트도 검색 결과 기준으로 변경
고객별 히스토리 목록도 검색 결과만 표시
```

### 지도 위치 누락

증상:

```text
일부 고객만 지도에 표시됨
```

가능 원인:

```text
상세 주소, 동호수, 괄호 설명, 지번 보조 정보가 OSM 검색에 혼입
```

개선:

```text
도로명주소 파싱 강화
지도 검색 주소를 도로명 + 건물번호 중심으로 정규화
표시 실패 고객 목록과 사유를 지도 하단에 표시
중복 주소 검색 제거
```

## 향후 검토할 기능

- 카카오맵 API 전환 검토
  - 무료 사용량
  - 도로명주소 정확도
  - JavaScript 키 관리
  - 도메인 등록
- SMS API 서버 기반 대량 문자 별도 프로젝트
- 사진 OCR 기반 고객리스트 추출
  - PWA에서는 웹 OCR 또는 파일 업로드 방식
  - Swift 앱에서는 Vision 프레임워크 활용 가능
- 다중 사용자/인증 기능
  - 현재는 개인 또는 소규모 테스트용 무인증 방식
  - 여러 사람이 같은 데이터를 공유하려면 서버와 인증 구조 필요
- iCloud Drive, Google Drive, Dropbox 등 외부 저장소 백업 연동

# ROADMAP

이 문서는 앞으로 개발할 기능과 검토할 개선사항을 관리하기 위한 목록입니다.

## 가까운 개선

- 모바일에서 카드형/목록형 전환 사용성 지속 점검
- 문자앱 실행 시 iPhone Safari, 홈화면 PWA, PC 브라우저별 동작 차이 확인
- Google Drive 동기화 후 새 기기 복원 흐름을 실제 사용자 계정으로 반복 테스트
- 완료/미완료, 터치 이력, 스케줄 카운트 기준을 화면별로 더 명확히 표시
- 지도 위치 누락 고객의 원인을 더 구체적으로 표시
- 고객 검색창 위치와 필터 조합 사용성을 실제 사용자 피드백으로 조정

## 중기 기능

- 삭제 동기화 설계
  - `deletedAt` 필드 도입
  - Drive 병합 시 삭제 상태 전파
  - 실수 삭제 복구 가능성 검토
- 고객 리스트 단위 백업/복원
  - 전체 백업 외에 특정 고객리스트만 내보내기
  - 특정 리스트만 Drive에서 복원
- 데이터 충돌 해결 화면
  - 같은 고객이 여러 기기에서 수정된 경우 비교
  - 사용자가 어느 값을 유지할지 선택
- CSV import 미리보기 강화
  - 연락처 열 후보 여러 개 표시
  - 생년월일, 주소 파싱 결과 미리보기
  - 저장 전 오류 행만 따로 표시
- 지도 정확도 개선
  - Kakao 지도 전환 또는 병행 검토
  - 주소 좌표 변환 캐시 관리
  - 위치 표시 실패 주소 재시도 기능

## 장기 검토

- 독립 iOS 앱 개발
  - Swift/SwiftUI
  - Vision 프레임워크 OCR
  - 네이티브 연락처/문자/지도 연동
- OCR 기반 고객리스트 인식
  - 출력물 사진 촬영
  - 텍스트 추출
  - 고객명/연락처/주소 후보 매핑
- 별도 서버 기반 다중 사용자 운영
  - 계정/권한 관리
  - 중앙 백업
  - 관리자 승인
  - SMS API 연동
- 단체 문자 고도화
  - iPhone PWA에서는 자동 개별 발송이 어려움
  - 별도 SMS API 서버 또는 네이티브 앱 프로젝트로 분리 검토

## 보류된 요구사항

- 사용자 개입 없는 SMS 자동 발송
  - iOS PWA 정책상 불가
- 문자 실제 발송 성공 여부 확인
  - PWA에서는 확인 불가
- 실제 통화 연결 여부 확인
  - PWA에서는 확인 불가
- 백그라운드 자동 동기화
  - iOS PWA에서 안정적으로 보장하기 어려움

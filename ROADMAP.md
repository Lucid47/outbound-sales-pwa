# ROADMAP

이 문서는 앞으로 개발할 기능과 검토할 개선사항을 관리하기 위한 목록입니다.

## 제품 개발 전략

1. 현재 PWA는 유지하면서 기능, 성능, 안정성을 계속 고도화한다.
2. OCR 기능은 PWA와 분리해 별도 기능으로 먼저 구현한다.
   - 1차는 Mac 기반 검증 도구로 개발한다.
   - Apple Vision OCR과 표 복원 품질을 먼저 확인한다.
   - 이후 iPhone/iPad 문서 스캔 입력으로 확장한다.
3. OCR 결과가 안정화되면 PWA의 고객리스트 import 흐름과 병합한다.
   - 우선은 OCR 도구가 만든 CSV를 PWA가 가져오는 방식으로 연결한다.
   - 가능하면 사용자가 PWA 안에서 OCR 기반 import를 자연스럽게 시작할 수 있게 한다.
4. PWA 병합이 기술적으로 어렵거나 사용자 경험이 부족하면 네이티브 앱으로 전환 개발한다.
5. 네이티브 앱으로 전환하는 경우 iPhone/iPad 우선 개발 후 Android 확장을 검토한다.
   - iPhone/iPad: VisionKit, Apple Vision
   - Android: ML Kit Document Scanner, ML Kit Text Recognition v2
6. 네이티브 앱 개발환경과 초기 프로젝트 준비는 `docs/native-app-setup.md`를 기준으로 진행한다.

## 가까운 개선

- 최우선: 가져온 고객 연락처 일괄 등록
  - 파일/사진/OCR/붙여넣기로 가져온 고객을 iOS 연락처에 일괄 생성
  - 기존 고객리스트에서도 `연락처 등록`을 실행해 나중에 일괄 등록 가능
  - 고객리스트 이름과 같은 연락처 리스트/그룹을 만들고 고객을 해당 그룹에 배정
  - 기본 옵션은 고객명 앞에 `#` 접두어를 붙여 저장
    - 카카오톡 자동등록 방지 목적의 기본값으로 사용
    - 사용자가 접두어 사용 여부를 켜거나 끌 수 있고 접두어 문구도 수정 가능
  - 이미 등록된 연락처와의 중복 검사 및 업데이트/건너뛰기 선택
  - 등록 전 미리보기와 등록 결과 요약 제공
- 모바일에서 카드형/목록형 전환 사용성 지속 점검
- 문자앱 실행 시 iPhone Safari, 홈화면 PWA, PC 브라우저별 동작 차이 확인
- Google Drive 동기화 후 새 기기 복원 흐름을 실제 사용자 계정으로 반복 테스트
- 완료/미완료, 터치 이력, 스케줄 카운트 기준을 화면별로 더 명확히 표시
- 지도 위치 누락 고객의 원인을 더 구체적으로 표시
- 고객 검색창 위치와 필터 조합 사용성을 실제 사용자 피드백으로 조정

## 중기 기능

- 삭제 동기화 설계
  - `deletedAt` 필드 도입
  - Drive 병합 시 삭제 상태 전파
  - 실수 삭제 복구 가능성 검토
- 고객 리스트 단위 백업/복원
  - 전체 백업 외에 특정 고객리스트만 내보내기
  - 특정 리스트만 Drive에서 복원
- 데이터 충돌 해결 화면
  - 같은 고객이 여러 기기에서 수정된 경우 비교
  - 사용자가 어느 값을 유지할지 선택
- CSV import 미리보기 강화
  - 완료: 저장 전 컬럼 매핑 확인, 헤더 없는 CSV의 열 직접 매핑
  - 남음: 연락처 열 후보 여러 개 표시
  - 남음: 생년월일, 주소 파싱 결과 미리보기
  - 저장 전 오류 행만 따로 표시
- 엑셀 파일 import
  - `.xlsx` 파일에서 첫 번째 시트 또는 사용자가 선택한 시트를 읽어 고객리스트로 변환
  - CSV와 동일하게 열 의미는 사용자가 지정할 수 있게 설계
  - iOS 네이티브에서는 Swift 기반 엑셀 파서 도입 또는 사전 CSV 변환 방식을 비교
- 지도 정확도 개선
  - Kakao 지도 전환 또는 병행 검토
  - 주소 좌표 변환 캐시 관리
  - 위치 표시 실패 주소 재시도 기능
- 네이티브 지도 고도화
  - Apple MapKit 기반 지도 탭을 우선 사용
  - 고객 주소를 좌표로 변환해 지도 핀 자동 생성
  - 고객 핀 선택 시 고객 상세/전화/문자/길찾기 액션 연결

## 장기 검토

- 선택 고객 방문 순서 자동 최적화
  - 구현 우선순위는 마지막 단계로 보류
  - 임의 고객 5명 선택 시 현재 위치/지정 출발지 기준으로 방문 순서와 동선을 추천
  - 1차는 Apple MapKit 이동시간 + 앱 내부 TSP 계산으로 검증
  - 실사용 정확도가 필요하면 TMAP 경유지 최적화 API 또는 VROOM/OSRM/Valhalla 기반 서버 검토
  - 고객별 체류시간 기본 10분 및 사용자 수정값을 예상 방문시각에 반영
- 독립 iOS 앱 개발
  - Swift/SwiftUI
  - Vision 프레임워크 OCR
  - 네이티브 연락처/문자/지도 연동
  - 개발환경 가이드: `docs/native-app-setup.md`
  - 현재 초안: `native/OutboundSalesCore`, `native/OutboundSalesNative`, `native/OutboundSalesiOS`
  - 완료: 기본 SwiftUI 탭 구조, CSV 텍스트/파일 import, 고객리스트/고객 로컬 JSON 저장, Apple MapKit 지도 탭, 고객 상세/수정, 전화/문자/길찾기, 히스토리, 오늘 스케줄, JSON 백업/복원, 주소 좌표 변환
  - 다음 단계: 실제 기기 QA, 엑셀 파일 import, OCR 실제 사진/문서 선택과 Vision OCR, Google Drive 계정 연동
- OCR 기반 고객리스트 인식
  - 스캔 이미지 파일 입력으로 Mac 코어 기능 먼저 검증
  - OCR 결과의 텍스트와 좌표를 이용해 행/열 복원
  - 사용자가 각 열 이름을 지정한 뒤 CSV 생성
  - iPhone/iPad는 VisionKit 문서 스캔과 Apple Vision OCR로 확장 검토
  - Android는 ML Kit Document Scanner와 Text Recognition v2로 확장 검토
  - 상세 계획: `docs/customer-list-ocr.md`
- 별도 서버 기반 다중 사용자 운영
  - 계정/권한 관리
  - 중앙 백업
  - 관리자 승인
  - SMS API 연동
- 단체 문자 고도화
  - iPhone PWA에서는 자동 개별 발송이 어려움
  - 별도 SMS API 서버 또는 네이티브 앱 프로젝트로 분리 검토

## 보류된 요구사항

- 사용자 개입 없는 SMS 자동 발송
  - iOS PWA 정책상 불가
- 문자 실제 발송 성공 여부 확인
  - PWA에서는 확인 불가
- 실제 통화 연결 여부 확인
  - PWA에서는 확인 불가
- 백그라운드 자동 동기화
  - iOS PWA에서 안정적으로 보장하기 어려움

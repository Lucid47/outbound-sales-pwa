# CHANGELOG

## 2026-07-14

- 원본 Excel/PDF 대조 기반 OCR 14페이지 회귀 검증 및 원근 보정
  - 362행 원본 Excel과 동일 레이아웃 PDF 14페이지를 기준 데이터로 사용하고 사진 14장을 페이지 1~14에 자동 대응
  - 배경 과다 노출, 종이 구김, 가로·세로·180도 방향, 비스듬한 촬영과 원근 왜곡을 포함한 HEIC 입력 검증
  - Vision 문서 영역 검출과 Core Image 원근 보정 결과를 원본 이미지 후보와 함께 평가해 좋은 쪽을 자동 선택
  - 사진 방향 전후에서 문서 영역을 검출해 EXIF 방향과 촬영 방향에 따른 문서 검출 편차를 보완
  - 전화·날짜·금액 등 본문 신호가 있는 첫 행은 형태가 달라도 헤더로 제거하지 않도록 자동 헤더 판정 강화
  - 한 셀에 여러 행의 전화·날짜·금액이 합쳐진 후보를 감점하고, 본문 신호가 온전히 반복되는 후보를 우선
  - 격자 보정이 정상 원본 후보보다 행을 잃으면 희소 주변 행 근거가 있는 경우에만 행 감소를 허용
  - 최종 14장 모두 PDF 페이지 순서와 동일하게 복원하고 362/362개 행을 원본 Excel 행에 정렬
  - 전체 가중 평균 행 문자열 유사도 0.9786, 개별 사진 최저 0.8902를 로컬 기준으로 기록

- 회전·병합 헤더가 포함된 고밀도 고객리스트 OCR 표 복원 보강
  - 0·90·180·270도 후보를 모두 OCR하고 본문 반복 구조, 헤더 판정, 행·열 충실도로 올바른 방향을 자동 선택
  - 기존 8열 선호 점수를 제거해 11열 이상의 표도 실제 반복 열 수로 복원
  - 병합되거나 여러 줄인 헤더보다 반복되는 본문 행의 x축 위치와 담당자 등 반복값을 우선해 열과 데이터 행을 결정
  - 본문보다 위에 있는 요약 숫자 행은 표 데이터에서 제외하고 여러 줄 주소·비고는 가까운 본문 행에 병합
  - 헤더에서 나란히 인식된 선행 표시 열과 담당자 열을 별도 열로 분리
  - 같은 열의 OCR 문장 박스가 세로로 겹치거나 바로 이어지면 다음 고객과의 중심 거리보다 셀 내부 줄바꿈을 우선해 이전 고객 셀에 병합
  - 담당자 등 기준 열 값 하나만 고립된 행은 기준값이 비어 있는 가장 가까운 고객 행으로 병합
  - 11열 표가 8열로 축소되지 않는 회귀 테스트 추가
  - 인쇄된 표의 긴 가로·세로 선을 기울기 허용 Hough 방식으로 검출하는 격자 분석기 추가
  - OCR 본문이 확정한 방향에서만 격자를 한 번 검출해 거꾸로 된 회전 후보가 격자 점수로 선택되지 않도록 구성
  - OCR 행·열 중심 사이의 실제 선만 셀 경계로 선택하고, 글자 획에서 생긴 과잉 선 후보는 배제
  - 격자 기반 결과가 기존 좌표 결과보다 구조 점수가 낮거나 행·열 수가 크게 달라지면 기존 방식으로 자동 전환
  - 임의로 생성한 4열×5행 표 이미지에서 격자 수를 검증하는 단위 테스트 추가
  - 표 선을 찾지 못한 사진도 반복 텍스트의 좌우 정렬로 최대 약 20도 기울기를 추정해 행을 평탄화
  - 원근과 종이 구김으로 열마다 행 높이가 달라져도 같은 개수로 반복된 열의 순서를 기준으로 행을 재정렬
  - 잘못 분리된 행을 헤더로 오인해 높은 점수를 주던 구조 평가를 수정하고 희소 행과 불규칙 행을 감점
  - 연속 번호가 있는 표는 증가 방향을 우선해 180도 뒤집힌 데이터 순서를 방지
  - 반복 본문 앞에 촬영된 전단지·배경 문자가 있으면 실제 헤더와 구분해 CSV 데이터에서 제외
  - 구김·원근·배경 혼입·90도 회전을 포함한 실제 사진 5장에서 각 19개 고객 행 복원을 로컬 검증

  - 파랑·초록·보라·주황·빨강·회색 계열을 선택하면 진행 순서에 따라 옅은색에서 짙은색으로 자동 배치
  - 선택한 계열 안에서 상태별 색상을 개별 조정하는 기존 기능 유지
  - 상태 개수, 색상 계열, 범례 표시 여부, 상태 이름·순서·개별 색상을 로컬 JSON과 Google Drive 전체·선택 백업에 포함
  - 기기간 동기화 시 수정 시각을 기준으로 최신 히트맵 설정을 병합하고 이전 백업 파일은 파랑 계열 기본값으로 호환

- Google Drive 반복 로그인 문제 수정
  - 최초 OAuth 연결에서 발급된 refresh token을 iOS Keychain에 기기 전용 정책으로 저장
  - access token이 유효한 동안 재사용하고 만료 시 refresh token으로 조용히 자동 갱신
  - 전체 동기화, 전체 저장, Drive 복원, 일반 백업 생성에서 매번 웹 로그인 세션을 열던 흐름 제거
  - 계정 표시 정보만 남은 기존 설치는 업데이트 후 한 번 재연결하도록 안내
  - Google 권한 철회로 refresh token이 무효화되면 연결 상태를 정리하고 재연결 안내

- Google Drive OAuth 로컬 설정을 Xcode 직접 빌드에도 자동 주입
  - 전용 스크립트를 우회한 빌드에서 Client ID가 비어 동기화가 비활성화되던 문제 수정
  - 실제 Client ID는 기존처럼 Git 제외 로컬 파일에만 보관

  - Shortcuts 전환 후 앱 자동 복귀 흐름과 점검 절차를 단축어 문서에 보강
